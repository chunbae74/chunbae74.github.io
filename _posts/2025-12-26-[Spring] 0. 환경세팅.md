## 0. 언어별 특징

> ### Node.js (Runtime)

**장점** <br>
    - 싱글 스레드 기반 비동기 처리에 최적화되어있음<br>
    - 채팅이나 실시간 알림 등 가벼운 요청이 많은 서비스에 강력<br>
    - npm 을 통한 다양한 라이브러리 사용 가능<br>

**단점** <br>
    - 복잡한 연산, 이미지 처리 등 CPU를 많이 쓰는 작업에 부적합<br>
    - 싱글 스레드의 한계

<br>

> ### Spring (Java/Kotlin Framework)

**장점** <br>
    - 안정성과 유지보수에 강함<br>
    - 멀티스레드 지원: 복잡한 연산, 대용량 트랜잭션 처리에 강함<br>
    - 강력한 보안 및 라이브러리<br>

**단점** <br>
    - 높은 진입장벽<br>
    - Node.js나 Python에 비해 메모리를 많이 사용 <br>
    - 서버 초기 구동 속도가 느린 편<br>

<br>

> ### Python (Django, FastAPI, Flask)

**장점**<br>
    - 문법이 직관적이고 쉬움<br>
    - AI 사용 및 데이터 수집시에 최고의 선택지<br>
    - FastAPI 덕분에 성능과 비동기 처리 능력도 크게 개선됨<br>

**단점**<br>
    - 실행 속도 자체는 느린 편<br>
    - 멀티 스레드 활용에 제약이 있음<br>

<br>

> ### Next.js (Full-stack Framework)

**장점**<br>
    - 서버 렌더링, 첫 페이지 로딩 속도 매우 뛰어남<br>
    - 라우팅, 빌드 최적화 등이 자동화되어 있어 별도 설정 필요없음<br>
    - 프론트엔드 로직과 백엔드 로직을 매우 긴밀하게 결합할 수 있음<br>

**단점**<br>
    - 무거운 백엔드 로직에는 부적합함<br>
    - 특정 플랫폼(Vercel)에만 최적화되어있음<br> 

<br><br>

## 1-2. 에디터 설정

> ### SDK 버전 확인

![Desktop View](/assets/img/[Spring]0.환경세팅/img02.png){: width="2029" height="755" }_사진 출처 : 코딩애플_

상단 메뉴에서 **파일 - project structure** 메뉴 얼어서 SDK가 프로젝트의 자바 버전 이상인지 확인하기.

<br>

> ### build tool 설정

![Desktop View](/assets/img/[Spring]0.환경세팅/img03.png){: width="2150" height="1405" }_사진 출처 : 코딩애플_

**파일 - Settings - gradle** 메뉴에서 Build and run 항목을 IntelliJ IDEA로 설정할 시,<br>**Settings - Java Compiler** 메뉴에 Additional command line parameters에 *-parameters*라고 추가해야함.
<br><br>
참고1. 프로젝트 폴더에 out폴터가 이미 있으면 안될 수 있으니 한 번 삭제 ㄱㄱ
<br>
참고2. 에디터에 Java Compiler 메뉴가 없으면 **build.gradle**파일 열어서
```java
compileJava {
    options.compilerArgs << '-parameters'
}
```
코드 추가해 주면 됨

## 2. 참고

> 프로젝트 파일들 설명

`src 폴더` 실제로 코드 짜는 곳<br>
`resources 폴더` HTML CSS Javascript 이미지 등 프론트 코드 짜는 곳<br>
`application.properties 파일` 환경변수나 프로젝트 설정 보관하는 곳<br>
`build.gradle 파일` 프로젝트 이름, 버전, 외부 라이브러리 등 기록용 파일<br>
`gradle 어쩌구~` 건드릴 일 없음 ㅇㅇ

<br>

> 변경사항 발견 시 자동 재실행

코드짠걸 매번 컴파일 할 수고스러움을 덜어주는 코드.<br>터미널 2개 오픈해준 뒤에 각각 터미널에 아래 코드 하나씩 입력.
```
// 실시간으로 빌드하라는 명령어
./gradlew build --continuous
```

```
// 프로젝트 실해하는 명령어
./gradlew bootRun
```
